# 内容管线种子表

> 定时触发 interview-writer 生成虚构故事素材的输入配置。
> 每次从"人设 × 主题 × 情绪弧"中抽取一组，跑出一篇结构化访谈稿。

## 使用方式

每次定时触发时：
1. 从下面三个维度各随机抽一条
2. 组合成一个 interview-writer 的启动 prompt
3. AI 同时扮演记者和受访者，跑完整访谈流程
4. 产出存入 `~/content work/content/interviews/auto-generated/`

## 组合示例

`P01 × T02 × A02` → 
> "一个在大厂干了 4 年的前端，受够了无意义的 CRUD，辞职后想用 AI 做自己的产品。一开始以为 cursor 能搞定一切，结果撞了无数墙。直到某天深夜跑通了第一个 demo，他才意识到：真正变了的不是技术，是他终于在为自己写代码。"

`P05 × T04 × A04` →
> "全职妈妈 3 年，每天围着孩子转，朋友圈全是辅食和早教。某天她发现自己连'我想做什么'都回答不了。她一直在逃避这个问题，直到孩子上幼儿园第一天，她坐在空荡荡的家里，终于跟自己说：你不能再等了。"

## 生成 prompt 模板

```
/interview 虚构故事生成

请你同时扮演记者和受访者，完成一次完整的 9 轮访谈。

受访者人设：{persona}
故事主题：{theme}
情绪弧线：{arc}

要求：
- 受访者的回答必须包含具体场景、具体数字、具体对话
- 不要泛泛而谈，每一轮都要有新的细节浮现
- 允许有犹豫、自相矛盾、说不清楚的地方（真实感）
- 最后产出标准的结构化访谈稿（故事线总览 + 素材卡片 + 金句清单 + 关键细节 + 情感转折点 + 未展开线索）
```
