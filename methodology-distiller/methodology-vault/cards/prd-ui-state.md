# PRD UI 状态卡

## 什么时候用

当 PRD 缺 UI 草图、页面结构、按钮说明或状态说明时使用。

## 解决什么缺口

补 PRD Detail 中的体验与 UI 行为。

## 操作步骤

1. 列出页面或视图。
2. 每个页面拆成 3-6 个主要区域。
3. 每个区域列出控件：按钮、输入框、选择器、菜单、列表、状态提示。
4. 每个控件写清动作和结果。
5. 每个页面补四类状态：空、加载、错误、成功。
6. 如果没有真实视觉稿，用文本草图表达布局。

## 文本草图格式

```text
页面：<名称>
┌─ 顶部区域：...
├─ 主操作区：...
├─ 内容列表 / 结果区：...
└─ 状态 / 提示区：...

主要按钮：
- 按钮名：点击后...

状态：
- 空状态：
- 加载：
- 错误：
- 成功：
```

## 不要做

- 不要把 UI 写成营销页文案。
- 不要只写“界面美观简洁”。
- 不要在没有必要时要求用户提供高保真设计。

## 质量门

实现者应该能根据文本草图搭出第一版页面，用户也能判断按钮和状态是否符合预期。

## 来源 / 借鉴

- Open Design Kit 的可操作方法表达。
- Material Design 控件和 Storybook 状态覆盖思想。
