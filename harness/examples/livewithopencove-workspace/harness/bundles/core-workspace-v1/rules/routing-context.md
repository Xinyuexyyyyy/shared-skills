# 路由与上下文规则

本组件决定当前输入应直接回答、调用 skill、使用工具还是进入完整流程，并控制规则和记忆的读取深度。

## 路由原则

1. 简单、一次性、可直接回答的请求不进入重流程。
2. 只有当前运行时真实可发现的 skill、工具和入口才能被选择。
3. 内部路由默认不展示；路线会改变交付物、权限或成本时，先用人话说明并取得确认。
4. 用户明确点名能力时优先使用，但仍需遵守权限、产物和验证边界。
5. 路由失败时报告缺失入口，不猜测一个看似相近的能力替代。

## 上下文加载顺序

1. 会话开始只读运行时入口和当前工作区规则，不预读私有记忆。
2. 用户说明任务后，先形成 Digest，再判断是否需要任务状态、历史决策或经验。
3. 接续已有任务时，先读任务索引和对应任务状态。
4. 接上次进度时，先读当前位置，再取时间线最近少量记录。
5. 涉及历史决策或避坑时，按主题检索对应文件，不整篇灌入。
6. 读到足以执行当前一步时立即停止。

## 防漂移

- 当前请求优先于旧会话、旧任务和旧记忆。
- 大文件按范围读取；经常变化的任务状态与长期稳定背景分开管理。
- 长会话在关键决策点重新锚定“当前目标、范围和禁止事项”。
- 路由表只保存意图到能力的映射，不复制领域 skill 的正文。
