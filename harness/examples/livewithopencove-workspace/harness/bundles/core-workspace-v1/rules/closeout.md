# 收尾规则

Closeout 让用户在任务完成、暂停、失败或阻塞时一眼看懂当前结果，不承担内部日志展示。

## 默认内容

1. 结果是什么。
2. 最短怎么验。
3. 哪些未完成、部分通过或未验证。
4. 只给与本轮相邻的下一步。

默认使用两到四个短段落。大事、失败或阻塞时使用 `结论 -> 怎么验 -> 未完成 -> 下一步` 四段。

## 状态规则

- 正常完成：给结果和当前证据。
- 部分完成：第一句明确“部分完成”，说明用户会受到的影响。
- 验证失败：不得包装成完成；给失败原因、影响和下一步。
- 阻塞：说明缺少什么决定、权限或外部条件。
- 用户叫停：说明已做、未做和恢复入口。

## 默认隐藏

路由详情、完整工具链、失败提示词、长日志、内部评分和记忆字段模板默认不展示。只有用户明确要求、任务本身在调试机制，或失败定位确实需要时才展开。

## 记忆边界

Closeout 可以提出一条可审阅的记忆候选，但不能直接写入长期记忆。候选必须说明写入价值和目标文件，用户确认后再由记忆制度处理。
