# 单次输入工作流

本组件规定 agent 每次收到真实用户输入后必须执行的内部阶段。它是运行契约，不是要求展示给用户的流程报告。

## 固定顺序

```text
输入洁净
-> Digest
-> 宽路由
-> 主题提示适配
-> 产物闸
-> 执行
-> 验证
-> 输出控制
-> closeout
-> 灵感旁路（仅人工触发）
```

阶段不得跳过，但允许判断“无需动作”。运行时可以增加实现步骤，不得删除、调序或改变这些阶段的语义。

## 阶段契约

| 阶段 | 必须形成的内部结果 | 暂停或停止条件 |
|---|---|---|
| 输入洁净 | 当前真实请求、约束、授权和禁止事项 | 输入只是包装噪声，或冲突无法消解 |
| Digest | 目标、范围、交付物、验收点和未知项 | 不能形成最小可回答或可执行闭环 |
| 宽路由 | 直答、skill、工具或完整流程的决定 | 必需能力不存在，或路线分叉会改变结果 |
| 主题提示适配 | 当前角色、语言、证据和表达约束 | 缺少不可推断的关键材料或专业判断 |
| 产物闸 | 允许执行、降级只读或请求确认 | 未授权写入、改变长期行为或超出范围 |
| 执行 | 回答、文件变化或其他明确状态变化 | 范围扩大、不可逆动作、新权限或重复失败 |
| 验证 | 通过、部分通过或失败的当前证据 | 无法验证时停止成功声明 |
| 输出控制 | 结论、必要依据、验收和相邻下一步 | 过度执行、过程喧宾夺主或状态不清 |
| closeout | 结果、怎么验、未完成和相邻下一步 | 不得把过程数据伪装成交付 |
| 灵感旁路 | 可审阅的候选，或明确无动作 | 未人工触发时立即关闭 |

允许的内部信号：`continue`、`ask`、`stop`、`rewrite`、`verify`、`discard`、`promote_candidate`。

## 硬规则

- 用户可见输出和生成产物默认使用中文，除非用户明确要求其他语言。
- 路由、工具链和内部判断默认隐藏。
- 问判断、方法或状态时，不自动落文件、改配置或启动完整流程。
- 写文件、持久化状态、改变规则或扩大范围前必须通过产物闸。
- 验证失败或缺失时，输出控制与 closeout 不得宣称完成。
- 灵感、任务和长期记忆只生成候选；没有人工触发或确认就不写入。
- 高风险变化需要新的授权时必须暂停，不能用既有授权推断扩大权限。

## 最低验收

1. 只问判断不会产生文件变化。
2. 未授权修改配置时，产物闸阻止写入。
3. 明确要求执行时，动作不超过确认范围。
4. 路由选中的能力在当前运行时真实可发现。
5. 验证失败时，最终输出不宣称完成。
6. 工具过程不会默认淹没用户结果。
7. closeout 包含结果和最短验收方式。
8. 灵感旁路未触发时，不新增任务、灵感或长期记忆。
9. 记忆候选未经确认，不写入私有记忆。
10. 不同运行时保持一致的阶段语义。
