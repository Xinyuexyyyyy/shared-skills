## 二、命名规则

1. **统一下划线，不用连字符混用**：`task_draft`（✅），不要又 `task_draft` 又 `task-draft`。
   - 历史遗留：DailyWork 的 `task-draft` 应并入 `task_draft`。
2. **统一英文 / 拼音，不用中文目录名**：中文目录命令行要转义、跨设备易乱码。
   - 历史遗留：`英语外刊`→`english-reading`、`opencove宣发参考资料`→`opencove-promo-refs`（迁移时再改，不强制）。
3. **项目目录用语义短名**：`dual-fuel-engine-papers` ✅，不用日期戳当目录名。
4. **不建单文件碎片目录**：`tmp/`、`sticky/`、`data/`（各只有 1 个文件）应并入 `task_draft/` 或对应项目。
5. **绝不留误建的字面量目录**：`~`、`Users` 这种是脚本 `~`/`$HOME` 没展开建出来的垃圾，发现即清。

## 四、从根上规避（误建目录的根因防御）

误建的 `~`/`Users` 目录不是手滑，是**脚本用了未展开的路径**。根因防御：

1. **路径里的 `~` 必须靠 shell 展开，别写进引号**：`cd ~/dir`（✅）；`mkdir "~/dir"`、`open('~/dir')`（❌ 会建出字面量 `~` 目录）。Python 用 `os.path.expanduser("~/...")` 或 `Path.home()`，不直接拼 `~`。
2. **写文件前确认 cwd**：相对路径在错误的 cwd 下跑，会把文件写到意外位置。skill/脚本落盘前用绝对路径，或显式 `cd` 到工作区根。
3. **绝对路径优先**：跨设备、跨 skill 调用时，用 `/Users/sure/<工作区>/...` 绝对路径，少用相对路径和 `~`。
4. **落盘后自检**：脚本产出后，检查有没有在 cwd 下意外建出 `~`、`Users`、`$HOME` 这类字面量目录；有就是 bug，当场修 + 清。
5. **closeout 顺带扫**：收尾时若发现工作区根下冒出 `~`/`Users`/`$*` 目录，提示用户这是路径 bug 的产物。

> 已修案例（2026-06-19）：study-research 的 `~/学习/...`（氢内燃机研究数据，已移入 task_draft/recovered-from-tilde/）、`Users/.../supervisor-scout/...`（0 字节空壳，已删，真数据在正确位置）。根因都是脚本路径没展开。

## 五、新建目录时的自检

1. 这个目录是不是已有标准一级目录能装下？（别新造同义目录）
2. 名字是下划线、英文、语义短名吗？
3. 是不是单文件碎片？（是 → 并进已有目录）
4. 路径里有没有未展开的 `~`/`$VAR`？（有 → 是 bug，别建）
